## API Key Authentication

All requests to the Gateway require an `X-Api-Key` header:

```bash
curl https://api.acg.ai/chat/completions \
  -H "X-Api-Key: acg_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4", "messages": [{"role": "user", "content": "Hello"}]}'
```

### Key Format

API keys follow the format: `acg_<24_hex_chars>`

Example: `acg_a1b2c3d4e5f6a7b8c9d0e1f2`

### Key Scopes

| Scope | Description |
|-------|-------------|
| `read` | Read-only access to data |
| `write` | Create and update resources |
| `admin` | Full administrative access |

## Creating API Keys

### Via Admin API

```bash
# Create organization
curl -X POST http://localhost:3002/v1/organizations \
  -H "Content-Type: application/json" \
  -d '{"name": "Acme Corp", "slug": "acme"}'

# Create project
curl -X POST http://localhost:3002/v1/projects \
  -H "Content-Type: application/json" \
  -d '{"name": "Production", "organizationId": "org_xxx"}'

# Create API key
curl -X POST http://localhost:3002/v1/api-keys \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production API Key",
    "organizationId": "org_xxx",
    "projectId": "proj_xxx",
    "scopes": ["read", "write"]
  }'
```

### Via CLI

```bash
acg auth login
acg auth create-key --name "Production"
```

## Rate Limits

Rate limits are enforced per API key:

| Tier | Requests/minute | Requests/day |
|------|----------------|--------------|
| Free | 10 | 1,000 |
| Developer | 60 | 10,000 |
| Startup | 300 | 100,000 |
| Business | 1,000 | 1,000,000 |
| Enterprise | Custom | Custom |

Rate limit headers are included in every response:

```
X-Rate-Limit-Limit: 60
X-Rate-Limit-Remaining: 45
X-Rate-Limit-Reset: 1640995200
```

## Multi-Tenancy

Each organization is completely isolated:

- Separate API keys
- Separate audit logs
- Separate compliance scores
- Separate billing
- No data leakage between orgs
