## Organization Hierarchy

```
Organization
в”ңв”Җв”Җ Project (dev, staging, prod)
в”Ӯ   в”ңв”Җв”Җ API Keys
в”Ӯ   в”ңв”Җв”Җ Policies
в”Ӯ   в””в”Җв”Җ Audit Logs
в”ңв”Җв”Җ Users
в”ңв”Җв”Җ Subscription
в””в”Җв”Җ Compliance Scores
```

### Organization
The top-level tenant. Has billing, subscription, and compliance settings.

### Project
A logical grouping within an organization. Each project gets its own API keys and audit logs.

### API Key
Authenticates requests. Scoped to a project with specific permissions.

## Kernel Components

The **Platform Kernel** (`@acg/kernel`) is the execution core:

| Component | Purpose |
|-----------|---------|
| **Plugin Runtime** | Dynamic plugin loading with lifecycle hooks |
| **Rule Engine** | ESLint-style rule evaluation |
| **Registry** | Centralized component registry |
| **Asset Graph** | Dependency tracking between AI assets |
| **Evidence Engine** | Tamper-proof audit trail with hash chain |

## Compliance Packs

| Pack | Region | Controls |
|------|--------|----------|

| HIPAA | US | Encryption, audit logging, PHI redaction |
| GDPR | EU | Consent, right to erasure, data portability |
| PCI-DSS | Global | Card masking, encryption, access control |
| SOC 2 | Global | Access control, encryption, monitoring |
| ABDM | India | ABHA verification, consent artifacts |
| AI Safety | Global | Prompt injection, jailbreak, data leakage |
| Banking | India | Data localization, fraud detection |
