## Overview

The `acg scan` command analyzes your codebase for AI-related security and compliance issues.

## Running a Scan

```bash
# Scan current directory
acg scan

# Scan specific path
acg scan /path/to/project

# Output as JSON
acg scan --json
```

## What Gets Scanned

| Category | Detection Method |
|----------|-----------------|
| **Secrets** | Regex patterns for API keys, tokens, private keys |
| **AI SDKs** | Import/require patterns for OpenAI, Anthropic, etc. |
| **Model References** | String patterns for GPT-4, Claude, etc. |
| **Environment Variables** | Known AI-related env vars |
| **Prompt Files** | Files matching prompt/template patterns |
| **Observability Configs** | Langfuse, LangSmith, Helicone configs |

## Scan Results Structure

```typescript
interface ScanResult {
  rootPath: string;
  filesScanned: number;
  findings: ScanFinding[];
  summary: {
    sdks: string[];
    promptsFound: number;
    secretsFound: number;
    envVarsFound: number;
    configsFound: number;
    modelRefs: string[];
    riskScore: number;
  };
}
```

## CI/CD Integration

```yaml
# GitHub Actions
- name: ACG Scan
  run: |
    npx @acg/cli scan --json > scan-results.json
    if [ $(cat scan-results.json | jq .summary.secretsFound) -gt 0 ]; then
      echo "Secrets detected! Failing build."
      exit 1
    fi
```
